# s-cool-backend

## Branch/PR について

### - 常設ブランチ
```
develop
- 本番環境 PR用branchはここから
```
### - 新規のブランチ
```
feature/#100_feature_button
fix/#200_fix_button
hotfix/fix_button_bug

命名規則
1.接頭辞は以下の通りとする.
  機能追加: feature
  修正: fix
  バグの修正: hotfix
2.issueが存在する場合は接頭辞に続けて #100 のようにissue番号を含める.(複数人での開発の際は極力issueを立てた後branchを立てるようにする).
3.最後に要約を記載 
```
